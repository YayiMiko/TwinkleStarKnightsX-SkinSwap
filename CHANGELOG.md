# 更新记录

## 未发布

- Android 安装和卸载现在会等待手机完成 USB 调试授权，并明确提示需要在手机上点击“允许”。
- 修复手机处于 `unauthorized` 状态时，PowerShell 将 ADB 提示误报为脚本异常并立即退出的问题。
- Android 构建过程会自动改用可写的纯英文临时目录，修复 Windows 用户名含中文时 `Illegal byte sequence` 导致安装失败的问题。

## Android Release 0.3.0 - 2026-07-17

- 修复多次查看动画、切换场景或进入战斗后，所有角色的通常攻击动画可能只剩背景和特效的问题。
- 改为每次播放时临时使用变身动画，播放完成后立即恢复游戏原有对象，不再永久改写游戏的动画表。
- 安卓运行时改用游戏自身的 Addressables 资源管理器请求变身骨骼，不再直接打开第二份 AssetBundle。
- 图鉴原生变身动画与通常攻击共享游戏的下载、去重和引用计数流程，避免连续切换后异步加载停滞。
- 资源尚未加载完成时，本次通常攻击会保留原演出；加载完成后再次播放即可正常替换，不会让界面卡死。

## Android Release 0.2.9 - 2026-07-17

- 安装器会自动查询友仓最新的闪耀星骑士兼容安装包，不再因游戏更新而尝试安装旧版本。
- 如果兼容安装包需要先更新，游戏会自动启动；在手机游戏内完成更新后，再次运行同一个 BAT 即可完成 MOD。
- ADB 从 Windows 临时目录运行，不再占用发布包文件夹；安装或卸载后可以正常移动、覆盖或删除文件夹。
- 已确认通常攻击 1 和 2、完整重启、汉化、卸载恢复及文件夹清理均正常。

## Android Dev 0.2.9 - 2026-07-17

- ADB 现在从 Windows 临时目录中的独立副本运行，不再占用解压出的测试包文件夹；即使 ADB 清理失败，测试包文件夹仍可删除或替换。
- 每次运行都会查询 `anosu/DMM-Mod` 的最新闪耀星骑士兼容安装包，不再固定使用某个旧游戏版本。
- 下载后核对 GitHub 公布的文件大小和 SHA-256，并继续检查应用包名、签名及关键组件；不会强制降级或卸载现有游戏。
- 若兼容安装包版本较新，第一次运行会先完成应用更新并启动游戏；在手机游戏内完成更新后，关闭游戏并再次运行同一个 BAT 即可完成 MOD。
- ADB 拒绝安装时会显示手机返回的具体原因，方便区分版本、签名和系统权限问题。

## Android Release 0.2.8 - 2026-07-14

- Android 正式版支持一键安装、更新和恢复，保留原有汉化及手机上的大型游戏数据。
- 已确认通常攻击 1 和 2 的动画替换有效，并完成连续击杀、场景切换、完整重启及卸载恢复验证。
- 安装中断后可以重新运行，已经下载或写入手机的有效资源会继续复用。
- 当前版本仍需 Windows 电脑和 USB 调试；无需电脑的安卓安装方式正在开发中。

## Android Dev 0.2.8 - 2026-07-14

- 撤回 0.2.7 使用游戏不支持的同步资源接口的无效方案，恢复旧版确认可用的异步加载方式。
- 恢复对 Bundle、异步请求、SkeletonData 和 Cutin 克隆的显式保留，修复资源加载启动后始终无法完成的问题。
- 保留新版的游戏版本校验、场景切换清理和单角色失败隔离；切换 Cutin 管理器后会释放上一场景保留的对象。

## Android Dev 0.2.7（已撤回）- 2026-07-14

- 同步加载方案调用了游戏未提供的 Unity 接口，无法替换动画，请勿继续使用该版本。

## Android Dev 0.2.6 - 2026-07-14

- Platform Tools 下载改用多个 Google 官方地址；连接失败或超时后自动切换，解决部分网络访问 `dl.google.com` 时无限等待的问题。
- 所有候选地址共享同一个固定 SHA-256，下载内容不匹配时不会执行。

## Android Dev 0.2.5 - 2026-07-14

- Windows ADB 升级到 Platform Tools 37.0.1；该版本更换了 Windows USB 后端，以规避已观察到的 37.0.0 原生崩溃。
- 资源安装器遇到 ADB 服务意外断开时会自动重启并重试一次；权限、授权或资源校验错误不会被误判为可重试故障。

## Android Dev 0.2.4 - 2026-07-14

- 安装和卸载 BAT 结束时会关闭随解压包启动的 ADB 服务，避免流程结束后 ADB 继续占用解压文件夹。
- 成功、取消或报错都会执行相同的 ADB 清理；关闭命令行窗口后即可删除旧版文件夹。

## Android Dev 0.2.3 - 2026-07-14

- 修复 Windows PowerShell 5 将 ADB 首次启动服务时的正常提示误判为卸载失败的问题。
- 安装与卸载入口现在会先启动 ADB 服务，并以实际退出码判断是否成功。

## Android Dev 0.2.2 - 2026-07-14

- 安卓卸载工具现在可以在全新解压目录中自动下载并校验 ADB、便携 Python 和恢复用兼容安装包，不再要求先从同一目录运行安装器。
- 安装与卸载共用同一套工具准备和兼容安装包校验逻辑，ADB 下载固定到已验证的具体版本。
- 恢复完成后会确认手机上的游戏已经重新启动；不会卸载应用或清除大型游戏数据。

## PC Release 1.2.2 - 2026-07-14

- PC 发布包改为直接携带已编译并校验的插件，普通用户不再下载或安装 .NET SDK。
- 移除普通用户侧的 SDK 版本检查，解决部分电脑反复提示 .NET SDK 6.0.428 安装失败的问题。
- 从旧版升级成功后会清理不再需要的便携式 .NET SDK、旧源码和构建脚本。

## Android Dev 0.2.1 / PC Dev 1.2.1 - 2026-07-14

- Android 只接受仓库明确列出的完整兼容安卓安装包 SHA-256，不再把公开证书视为整包信任依据。
- Android 固定并校验 ADB、Python、构建工具和签名工具，下载中断不会留下被继续执行的半成品。
- Android 会在安装 Hook 前核对游戏版本和目录表 SHA-256；游戏更新后旧映射自动停用。
- Android 改为先原子写入全部资源和映射，最后覆盖安装 APK 并确认游戏已启动。
- Android 在 Cutin 管理器切换时释放旧 Clone 与变身资源，加载失败或超时后可再次尝试。
- Android 复用手机缓存时同时检查文件大小和 UnityFS 文件头，推送后核对完整 SHA-256。
- PC 角色注册发生瞬时错误时允许重试，连续三次失败后才在本次游戏进程中停用。

## Android Dev 0.2.0 - 2026-07-13

- 新安装器可自动取得公开的标准兼容安卓安装包，将汉化与本 MOD 一起内置后通过 ADB 覆盖安装。
- 覆盖安装不会卸载游戏或清除大型游戏数据；资源安装仍优先复用游戏缓存和 MOD 已下载内容。
- 下载的兼容安卓安装包、签名工具和游戏资源均校验来源与 SHA-256，不包含在本仓库或 Release 压缩包中。
- 卸载时会覆盖恢复未内置本 MOD 的兼容安卓安装包，默认保留变身资源以便再次安装。
- 已在 Android 16 真机通过首页、汉化、通常攻击 1 和 2、连续击杀、图鉴预览后战斗、场景切换、完整重启、资源补齐及卸载恢复测试。

## PC Release 1.2.0 - 2026-07-13

- 卸载工具新增“彻底卸载”和“暂时停用”两个明确选项。
- 彻底卸载会清理插件、配置、下载资源、本地工具和 MOD 文件夹；仅在确认没有其他插件使用时移除由本 MOD 安装的 BepInEx。
- 卸载前检查游戏进程，失败时显示明确提示，避免插件仍被占用而产生残留。
- PC 端改为只下载实际使用的变身 Bundle，首次下载量由约 2 GB 降至约 1 GB，并移除 UnityPy 和 synthetic atlas 生成流程。
- 更新改为暂存、验证、带回滚安装；下载或编译失败时不再删除原本可用的资源与配置。
- 插件会核对游戏程序、IL2CPP metadata 和 Catalog 指纹，游戏更新后旧映射会安全停用并提示重新应用。
- 同一角色快速重复触发时按顺序处理覆盖请求，并限制运行日志大小和动画别名缓存生命周期。
- 已完成通常攻击 1、通常攻击 2、连续击杀、图鉴预览后进入战斗、场景切换和重启游戏的实机验证。

## PC Release 1.1.3 - 2026-07-13

- 安装器现在会等待 BepInEx 完成兼容文件验证；已有文件仍然有效时可直接继续安装。
- 已在当前游戏版本完成资源更新、插件编译和实际运行验证。
